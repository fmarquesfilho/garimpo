# T-0028: Configurar coleta no scheduler (popular snapshots do zero)

| Campo | Valor |
|-------|-------|
| Status | `done` |
| Prioridade | alta |
| Estimativa | M |
| Sprint | 2026-S27 |
| Tags | [backend, scheduler, collector, bigquery, validação] |

## Descrição

BigQuery foi resetado. O scheduler precisa ser configurado com cron jobs
para popular snapshots novamente. A coleta deve garantir que o mesmo
produto_id apareça em múltiplos snapshots (para que variações sejam detectáveis).

## Critérios de aceite

- ✅ "Scheduler recebe SetSchedule via gRPC com crons das buscas ativas"
- ✅ "Após primeira coleta, snapshots contêm produto_ids repetidos entre dias"
- ✅ "Analyzer /quedas retorna variações reais após 2+ coletas"
- ✅ "Endpoint /api/lojas/novidades mostra produtos novos detectados"
- ✅ "Validar via query: SELECT produto_id, COUNT(*) FROM snapshots GROUP BY 1 HAVING COUNT(*) > 1"
