## Descrição

O publisher Go atualmente lê TELEGRAM_BOT_TOKEN e TELEGRAM_CHAT_ID de env vars
globais. Com multi-tenancy, cada tenant configura seus próprios tokens via
onboarding (/api/onboarding/telegram). O publisher precisa usar os tokens
do tenant ao publicar, não tokens globais.

## Critérios de aceite

- ✅ "C# API envia token e chat_id do tenant junto no request gRPC (Publish)"
- ✅ "Publisher Go aceita tokens no PublishRequest (campos opcionais no proto)"
- ✅ "Se token vem no request → usa ele; se não → fallback para env var (retrocompat)"
- ✅ "Tokens nunca logados em texto plano"
- ✅ "Frontend /canais permite configurar destino Telegram com token+chat_id por tenant"
- ✅ "Frontend /canais permite configurar destino WhatsApp com phone_number_id + token Meta"
- ✅ "Fluxo testado: onboarding → configurar telegram → publicar → mensagem chega"
